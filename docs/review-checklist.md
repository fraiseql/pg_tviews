## Pre-Review Preparation

### Content Validation
- [ ] **Technical accuracy**: All information verified against code
- [ ] **Completeness**: No missing information or TODO placeholders
- [ ] **Examples tested**: All code examples run successfully
- [ ] **Links valid**: All internal and external links work
- [ ] **Spelling/grammar**: Spell check and grammar review completed

### Style Guide Compliance
- [ ] **Formatting**: Follows [style guide](style-guide.md) standards
- [ ] **Terminology**: Uses consistent terms from style guide
- [ ] **Structure**: Proper heading hierarchy and document organization
- [ ] **Code blocks**: Proper language identifiers and formatting
- [ ] **Tables/lists**: Consistent formatting and alignment

## Content Review

### Clarity and Readability
- [ ] **Audience appropriate**: Right level of detail for target audience
- [ ] **Language clear**: No jargon without explanation
- [ ] **Active voice**: Uses active voice consistently
- [ ] **Concise**: No unnecessary words or repetition
- [ ] **Scannable**: Short paragraphs, clear headings, bullet points

### Structure and Organization
- [ ] **Logical flow**: Information flows logically from basic to advanced
- [ ] **Table of contents**: Present and accurate (for documents >500 lines)
- [ ] **Cross-references**: Appropriate links to related content
- [ ] **Header complete**: Version and date current
- [ ] **Sections balanced**: No sections too long or short

### Technical Content
- [ ] **Code examples**: Realistic data, expected output shown
- [ ] **Error messages**: Actual error text, not paraphrased
- [ ] **Version notes**: Version requirements clearly stated
- [ ] **Limitations**: Known limitations documented
- [ ] **Best practices**: Appropriate recommendations included

## Quality Assurance

### Testing
- [ ] **Examples executable**: All code examples can be run
- [ ] **Output verified**: Example output matches actual results
- [ ] **Edge cases**: Error conditions and edge cases covered
- [ ] **Prerequisites**: Required setup clearly documented
- [ ] **Cleanup**: Examples include proper cleanup

### Accessibility
- [ ] **Alt text**: Images have descriptive alt text
- [ ] **Color coding**: Not relied upon for important information
- [ ] **Screen readers**: Content accessible without visual formatting
- [ ] **Keyboard navigation**: Links and references keyboard accessible

### Maintenance
- [ ] **Version current**: Document version matches current release
- [ ] **Date updated**: Last updated date is current
- [ ] **Dependencies**: No broken dependencies on other docs
- [ ] **Templates used**: Appropriate templates followed
- [ ] **Standards followed**: All style guide rules applied

## Final Checks

### Documentation Standards
- [ ] **Template compliance**: Used appropriate template for document type
- [ ] **Naming conventions**: File and section names follow conventions
- [ ] **Directory structure**: File in correct location
- [ ] **Metadata complete**: All required metadata included

### Review Process
- [ ] **Self-review**: Author reviewed their own work
- [ ] **Peer review**: At least one other person reviewed
- [ ] **Feedback addressed**: All review feedback incorporated
- [ ] **Final proofread**: Final read-through for quality

---

## Review Process

### For Authors
1. Complete this checklist before requesting review
2. Fix all issues marked "No"
3. Request review from appropriate reviewer
4. Address all review feedback

### For Reviewers
1. Use this checklist during review
2. Mark specific issues found
3. Suggest improvements, don't just criticize
4. Verify technical accuracy
5. Check style guide compliance

### Approval Criteria
- [ ] All checklist items marked "Yes"
- [ ] No critical issues remaining
- [ ] Technical accuracy verified
- [ ] Style guide compliance confirmed
- [ ] Ready for production use

## Common Issues to Watch For

### Content Issues
- Outdated information
- Missing prerequisites
- Incomplete examples
- Broken links
- Inconsistent terminology

### Style Issues
- Inconsistent formatting
- Poor heading hierarchy
- Long paragraphs
- Missing table of contents
- Incorrect code block languages

### Technical Issues
- Untested code examples
- Wrong function signatures
- Missing error conditions
- Incorrect version information
- Broken cross-references

### Quality Issues
- Typos and grammar errors
- Unclear explanations
- Missing context
- Poor organization
- Accessibility problems
